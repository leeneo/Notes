之前在 VS code 出来的时候用过 VS code 没用过 atom。现在坚持用的 vim ，和正在学习 emacs.1. 为什么不用 vs code 或者 Atom ？答：有 sublime text 这个神器为什么还要考虑这两货？我只用过 sublime 写网页过前端的代码，以前配的 zen coding 插件，流畅的一逼（现在好久没用了）。而且个人觉得 sublime 在舒适性上面是要优于 atom 的（某次冲着这货是 Github 官方出品用了一下，就一下，就再也不想用了）。不用 vs code，是因为它和 sublime 重合太严重（至少在我的使用范围内是这样，比如写网页），我就不想浪费磁盘空间再去装个 vs code 了。而且 sublime 免费（尽管另外两个也免费）。2. 为什么之后不用 sublime ?答：我在 sublime 里面也是设置的 vintage（不知道拼对了没有），我喜欢这种 vim 快速的感觉。我使用的电脑是 mbp ，所以 mac 自带的一些快捷编辑方式 sublime 下也支持。而且 sublime 的插件功能太强大了！！！最后弃用 sublime 也是因为插件功能强大的原因。那些插件很让人学会了偷懒，某次习惯了用 zen coding 插件写网页代码，之后再在同学的电脑写个网页的代码，我居然写不出来了！插件为我省去了太多的细节，在不追求效率（毕竟没有工作压力，还是学习阶段）的情况下，一句一句敲可能更好吧！这也是我放弃 sublime 的原因。3. 关于 vim 的使用？答：真的，我喜欢那种纯文字的终端界面，我感觉很酷。也正是这个原因，我选择使用了 vim 而不是 macvim 。我觉得 vim 很酷，它很快，而且我觉得它是一种思维方式 ---- 用最小的动作去完成最多的事。所以就爱上了 vim，不能自拔。有时候还有点杞人忧天（vim 不像 emacs 有基金会支持，万一哪天这个软件没人维护了，该怎么办？天哪，这么优秀的软件，这样的事坚决不能发生，所以我一定要学会，并且安利给其它人）。当然，在我的范围内（主要是写 php ，js，C/C++），vim 绝对够用了。4. 关于 emacs 的使用？答：前面提到 vim 是种思维方式，那我觉得 emacs 是种生活方式。我觉得刚开始觉得 emacs 每次都要按住 ctrl 键就觉得 emacs 有点怪，之后尝试着编辑文件，我居然不知道最后怎么退出。各种组合键一脸懵逼。天哪，这太难了，我一定要学。后来就慢慢的照着网上的帖子安利去接受 emacs，挺不错的，越学越喜欢，越学越喜欢，你知道，感觉这种东西是玄学，但 emacs 是那种能让人上瘾的东西，比 vim 的魔力还要大。
